remix

regular expression alternation for tokenizers

What is this?

This allows joining regular expressions mainly for the purpos of constructing tokenizers.

Each regular expression can be named and have sub-named expressions which are tracked by namespaces with configurable delimiters. ReMix object can also compose other ReMix objects.

Regular expressions can be composed by simple templates.

Any regular expression with the same flags are joined together into a single | expression as an optimization.

The main exec()

Namespaces

If a ReMix object is given a name, all it's children are prefixed with that name and the nsDelimiter. If a regular expression with a namespace is matched, it's namespace is returned as the second element of the Match Array.

Mixing


/* unnammed prositional only matches */
var re = new Re([/foo/, /bar/]);

/* does not combine */
var re = new Re([/foo/, /bar/i]);

Named matches

  • example 1

var Re = require('remix').ReMix;

/* named match */
var re = new Re('foo', /foo/, /bar/);

var str = 'foobar';
/* [['foo'], 'foo', 0, 3] */
console.log(re.exec(str));
/* [['bar'], 'foo', 1, 6] */
console.log(re.exec(str));
  • example 2

/* embedded matches are namespaced with parent name */
var re = new Re('foo', {foo: /foo/, bar: /bar/});

var str = 'foobar';
/* [['foo'], 'foo.foo', 0, 3] */
console.log(re.exec(str));
/* [['bar'], 'foo.bar', 1, 6] */
console.log(re.exec(str));


/* alternative syntax */
var re = new Re('foo', [/foo/, /bar/]);

/* embedded named matches */
var re = new Re({foo: /foo/, bar: /bar/});

Methods

register (static)

Syntax

Spec spec = RegExp re | String template
ReMix ReMix.register((String name, Spec spec) | { name1: Spec spec1, ... })

Description

Register a template for later composition. Arguments are in one of two formats. Both forms specify name/spec pair(s). A name is a simple identifier which should match /^\w+$/. The spec can be either a regular expression or or another template.

Example

    ReMix.register('eol',        /(?:\r\n?|\n|\f)/);
    ReMix.register('emptyLines', '{eol}{eol+}');

    ReMix.register({
      cMultiComment: /\/\*[^*]*\*+(?:[^\/*][^*]*\*+)*\//,
      cLineComment:  "//{notEol*}{eol}"
    });

exec

Syntax

Array match = remix.exec(String string)

Description

Takes a string to match on and returns an array representing the results. See Matches

Example

var re = new ReMix('foobar', /foo(bar)/, { bar: /bar/, baz: { boo: /boo/ } });
var str = "foobarboobarfoobar", match;

match = re.exec(str); // [ [ 'foobar', 'bar' ], 'foobar',         0,  6 ]
match = re.exec(str); // [ [ 'boo'           ], 'foobar.baz.boo', 2,  9 ]
match = re.exec(str); // [ [ 'bar'           ], 'foobar.bar',     1, 12 ]
match = re.exec(str); // [ [ 'foobar', 'bar' ], 'foobar',         0, 18 ]
match = re.exec(str); // false

add(addArg)

Syntax

ReMix remix.add(addArg)

Description

Add the given specification. If a namespace is set, all children in this spec will be prefixed when resolved. All resolution happens only once and on demand.

Example

var re = new ReMix();
re.add(/regexp/);       // RegExp
re.add(function () {    // ReMix~SetupCallback
 return {               // ReMix~NamedRegExp
   name: "foo"
   spec: [              // ReMix~Spec[]
     {                  // ReMix~Pairs
       baz: 'foo{eol}$' // ReMix~Template
     },
     function () {      // ReMix~SetupCallback
       return /bar/
     }
   ]
 };
});

hasSpecs

Syntax

Boolean hasSpecs()

Description

Returns a boolean indicating if the current instance has any regular expression.

Example

var re = new ReMix();

console.log(re.hasSpec()); // false

re.add(/foo/);

console.log(re.hasSpec()); // true

re.clear();

console.log(re.hasSpec()); // false

compile

Syntax

Array regexn = remix.compile(Boolean force)

Compiles the current specification and returns the array of regular expressions generated. No need to call this yourself unless you are debugging or testing. This is called from exec() and toString(). compile() will cache the results unless force is true.

  • options([opt]) - If options are specified, they are merged with the current options. Returns the current options object.
  • name([name]) - Sets or retrieves the current name. The name is used to build the full namespace of each match.
  • clear() - This clears all add specifications and resets compilation. This effectively puts the object in a pristine state.
  • toString() - This is for debugging purposes and should not be used for matching.

Options

  • nsDelimiter - Sets the delimiter to use for named expressions to create namespaces (default .).

  • defaultFlags - These are the default flags all regular expressions are constructed with. (default g).

Matches

Templates

Templates are a very simple way to compose regular expressesion. Templates can be used in the place of regular expressions. You can register new templates or use the existing predefined templates. Here is a list:

  • hspace - Horizontal space, including unicode horizontal space code points

  • noHspace - hspace negated

  • vspace - Vertical space character class

  • number - General number match, includes all Javascript supported number formated

  • noVspace - Negated vspace character class

  • space - /\s/

  • noSpace - /\S/

  • word - /\w/

  • noWord - /\W/

  • any - Match any one character, doesn't depend on RegExp flags

  • eol - Match EOL on any platform

  • notEol- Not EOL character class

  • end - END of string or line depending on RegExp flags

  • begin - Beginning of line or string depending on RegExp flags

Template Example

Re.register('matchFoo', /foo/);
var re = new Re('foo', 'matchFoo');

var re = new Re('bar', '{matchFoo*}|bar');

Re.register('matchFooOrBar', '{matchFoo}|bar');
var re = new Re('bax', '\\s*{matchFooOrBar+}\\s*');
